In the Overview page, under the **“Equipment Health Snapshot”** section (currently displaying a single machine card with the “Live” indicator), enhance the component into a fully dynamic and interactive machine carousel/viewer.

### Functional Requirements

* Add **Previous** and **Next** navigation buttons/icons on the left and right sides of the machine snapshot card.
* Users must be able to cycle through all available machines using these controls.
* Navigation should wrap seamlessly:

  * Clicking **Next** on the last machine should move to the first machine.
  * Clicking **Previous** on the first machine should move to the last machine.

### Dynamic Machine Data Binding

When switching between machines, the entire snapshot content must update dynamically based on the selected machine’s live data object/state, including:

* Machine image
* Machine name/ID
* Operational status
* Health score
* RUL (Remaining Useful Life)
* Vibration
* Temperature
* Pressure
* Load
* Any associated trend indicators, charts, badges, or sparklines

Do not use hardcoded values inside the snapshot card.

Ensure all displayed values are sourced from the existing live simulation hooks/data structures already implemented in the project.

### Image Upload / Local File Support

Each machine card should support a customizable machine image.

Requirements:

* Add an option/button to upload or change the machine image from local files.
* Users should be able to select an image from their device for each machine individually.
* The uploaded image should immediately preview inside the machine snapshot card.
* Preserve the uploaded image per machine while navigating between machines.
* Use local component state, context, or persisted browser storage (preferred: localStorage) so uploaded images remain associated with the correct machine.
* Support common image formats (`.png`, `.jpg`, `.jpeg`, `.webp`).
* Include graceful fallback behavior if no custom image exists.

### UI / UX Requirements

* Maintain the existing UI styling, spacing, responsiveness, dark/light theme compatibility, and “Live” badge behavior.
* Add smooth animated transitions when changing machines.
* Ensure navigation buttons are visually clean and accessible.
* Keep the implementation modular and reusable.
* Prevent layout shifts when changing machine content or images.

### Optional Enhancements

* Add pagination dots or an index indicator below the card showing the current machine position.
* Add keyboard navigation support using left/right arrow keys.
* Add autoplay rotation mode with pause-on-hover behavior.
